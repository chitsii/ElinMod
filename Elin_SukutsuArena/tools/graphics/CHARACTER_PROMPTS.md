# キャラクター画像生成プロンプト集

Antigravity / Nano Banana 用のキャラクター画像生成プロンプトです。
ロア設定に基づいて作成しています。

---

## アートスタイルについて

### 基本スタイル
**村山竜大 (Ryuta Murayama)** - ファンタジーRPGイラストレーター
- 特徴: 繊細な線画、ダークファンタジー、重厚な装備描写、劇画調
- 参考: ドラゴンズドグマ、ファイナルファンタジータクティクス系

### ボス用追加スタイル
**ズジスワフ・ベクシンスキー (Zdzisław Beksiński)** - シュルレアリスム画家
- 特徴: 悪夢的、骨と肉の融合、荒廃した風景、不気味な美しさ
- 適用: アスタロト、カインの亡霊、ヌル、影の自己

---

## 共通プロンプト設定

### ポートレート用
```
背景: solid magenta background (#FF00FF)
構図: portrait, upper body, facing viewer
品質: highly detailed, masterpiece quality
```

### キャラチップ（スプライト）用
```
背景: solid magenta background (#FF00FF)
構図: full body, standing pose, game sprite style
品質: pixel art friendly, clean edges, no antialiasing
```

## 使用上の注意

### 1. 背景について
全ての画像は **マゼンタ背景 (#FF00FF)** で生成してください。
`process_graphics.py` でクロマキー処理を行い、透過PNGに変換します。

> [!IMPORTANT]
> クロマキー許容値 `COLOR_TOLERANCE = 100` に設定済み。
> 多少の色ムラがあっても透過されますが、キャラクターにマゼンタ系の色は使わないでください。

### 2. 推奨スタイル: ピクセルアート
高解像度イラストは縮小時に細部が潰れやすいため、**最初からピクセルアートスタイルで生成**することを推奨します。

```
推奨キーワード:
- Pixel art style, 16-bit, retro game sprite
- Clean pixel edges, no anti-aliasing
- Visible pixels but refined
```

### 3. 構図について

#### ポートレート
- 上半身、縦長構図
- 最終サイズ: 240x320px（高さ基準でリサイズ→中央クロップ）
- 正方形の生成画像から縦長に切り出されるため、**キャラクターは画像中央に配置**すること

#### スプライト（キャラチップ）
- **全身必須**（頭から足先まで全て描画）
- `FULL BODY VISIBLE, from head to feet` をプロンプトに含めること
- 最終サイズ: 128x128キャンバス内に83x104pxで配置
- Autocrop処理により余白は自動除去されるため、余白は気にしなくてOK

### 4. ポートレート先行ワークフロー（推奨）
キャラクターの一貫性を保つため、以下の手順を推奨します：

1. **ポートレートを先に生成**（キャラデザインの基準となる）
2. **生成したポートレートを参照画像として入力**し、スプライトを生成
   - `generate_image` の `ImagePaths` パラメータにポートレート画像のパスを指定
   - プロンプトに `Character based on provided image` を追加
3. これにより顔立ち・配色・装備が統一される

### 5. 生成後の処理
1. 生成画像を `Texture/` または `Portrait/` フォルダに配置
2. ファイル名を `{キャラID}.png` に変更
3. `uv run --with pillow python tools/graphics/process_graphics.py` を実行
4. 以下の処理が自動で行われます：
   - クロマキー透過 (tolerance=100)
   - **Autocrop** (余白除去) ※スプライトのみ
   - リサイズ (Portrait: LANCZOS, Sprite: NEAREST)
   - キャンバス配置 (Portrait: 中央クロップ, Sprite: 中央下配置)

### 6. スタイルのバランス
- **村山竜大風**: 繊細な線画、装備の細部、劇画的な表情
- **ベクシンスキー風**: 追加要素として使用。メインの形は村山風を維持しつつ、
  不気味な要素（骨、腐敗、融合、歪み）を付加
- **ピクセルアート**: ゲーム内での視認性を重視する場合に使用

### 7. よくある問題と対処

| 問題 | 原因 | 対処 |
|------|------|------|
| 背景が透過されない | マゼンタに近い色がキャラに含まれる | キャラの配色を変更 |
| スプライトが潰れる | 高解像度イラスト→極端な縮小 | ピクセルアートで生成 |
| キャラが切れている | スプライトで全身が描画されていない | `FULL BODY VISIBLE` を強調 |
| ポートレートで顔が見切れる | 正方形→縦長クロップで端が切れた | キャラを中央に配置 |
