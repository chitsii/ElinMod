# 仕様メモ (Design Notes)

## 飢え (Hunger) 仕様
`StatsHunger` クラスの定義より、飢えの段階（Phase）は以下の整数値定数で管理されている。値が大きいほど空腹度が深刻であることを示す。

*   **Bloated (食べ過ぎ)**: 0
*   **Filled (満腹)**: 1
*   **Normal (通常)**: 2
*   **Hungry (空腹)**: 3
*   **VeryHungry (飢餓)**: 4
*   **Starving (餓死寸前)**: 5

*   **判定ロジック**: `GetPhase()` は `value` (0-100) に基づいて計算される。
*   **注意点**: 空腹度が進む（腹が減る）につれて Phase の値は**上昇**する。 (2 -> 3 -> 4 -> 5)

## 眠気 (Sleepiness) 仕様
`StatsSleepiness` クラスの定義より、眠気の段階は以下の整数値定数で管理されている。値が大きいほど眠気が強い。

*   **Normal (通常/定義なし)**: 0 (推測: 定数が定義されていないが、初期状態や閾値未満として扱われる)
*   **Sleepy (眠気)**: 1
*   **VerySleepy (強い眠気)**: 2
*   **VeryVerySleepy (深刻な眠気/睡眠不足)**: 3

*   **判定ロジック**: 眠気が強くなるにつれて Phase の値は**上昇**する。

## Stats 共通仕様
*   **max**: 100
*   **min**: 0
*   **GetPhase()**: 現在値 (`value`) を `max` で割った割合 (0.0 - 1.0) を 10倍して 0-9 のインデックスに変換し、参照元の `phase` 配列から段階値を取得する。
*   **TrackPhaseChange**: `StatsHunger` や `StatsSleepiness` などで true に設定されており、段階変化時にメッセージ通知やイベントが発生する仕組みになっている。
